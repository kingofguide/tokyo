# 🎲 King of Tokyo — Monster Guide

A static web guide for King of Tokyo monsters. Built with plain HTML/CSS/JS + Markdown content files.

---

## Project Structure
```text
king-of-tokyo/
├── monsters/             ← Your content lives here (one .md file per monster)
│   ├── gigazaur.md       ← The classic lizard
│   ├── kraken.md         ← Ancient sea beast
│   ├── meka-dragon.md    ← Robotic terror
│   ├── cyber-bunny.md    ← High-tech rabbit
│   ├── space-penguin.md  ← Galactic flightless bird
│   └── alienoid.md       ← Extraterrestrial invader
├── build.py              ← Build script: converts .md files → data.json
├── data.json             ← Generated by build.py (don't edit manually)
├── index.html            ← The web app (no framework, just HTML/CSS/JS)
├── package.json          ← Optional: for node-based build (advanced)
└── README.md             ← This file
```

## Workflow: Adding or Editing a Monster

### 1. Create (or edit) a Markdown file in `/monsters`
Copy an existing file and rename it (e.g., `the-king.md`). The filename becomes the monster's ID. Each file requires **Frontmatter** (between the `---` lines) for structured data:

```markdown
---
name: Cyber Bunny
icon: 🐰
tagline: "Fast, sneaky, impossible to pin down."
difficulty: Intermediate
playstyle: Hit & Run / VP Racer
expansion: Base Game
complexity: 2
stats:
  aggression: 3
  resilience: 2
  energy: 4
  speed: 5
---

## Overview
Cyber Bunny is all about speed...
```

### 2. Run the build script
```bash
python3 build.py
```
This reads all `.md` files and generates `data.json`. You must do this every time you change a monster file.

### 3. Preview locally
Open `index.html` in your browser or use a local server:
```bash
python3 -m http.server 8080
```

## Monster File Reference

### Required Frontmatter Fields
| Field | Example | Notes |
| :--- | :--- | :--- |
| **name** | King Lizard | Display name |
| **icon** | 🦎 | Emoji shown in sidebar |
| **tagline** | "The classic..." | Short one-liner |
| **difficulty** | Beginner | Beginner / Intermediate / Advanced |
| **playstyle** | Tank | Short descriptor |
| **expansion** | Power Up! | Which box the monster is from |
| **complexity** | 1 | Numeric rating of how hard they are to pilot |
| **stats** | (see example) | aggression/resilience/energy/speed (1–5) |

## How to Contribute

We are looking for fans to help fill out the strategy guide! If you have a favorite monster or a winning tactic, please jump in.

1.  **Fork the Repository:** Create your own copy of the project on GitHub.
2.  **Add/Edit Content:** Update the `.md` files in the `/monsters` folder with your insights (see the "Suggested Sections" in existing files for inspiration).
3.  **Run the Build:** Ensure you run `python3 build.py` so your changes appear in the `data.json` file.
4.  **Submit a Pull Request:** Send your changes back to us for review!

**Need help with:**
* Detailed strategy tips for the "Mid Game" and "Late Game."
* Counter-play advice (how to beat specific monsters).
* Correcting any rule inaccuracies.
