# King of Tokyo — Monster Guide

A static web guide for King of Tokyo monsters. Built with plain HTML/CSS/JS + Markdown content files.

## Project Structure

```
king-of-tokyo/
├── monsters/             ← Your content lives here (one .md file per monster)
│   ├── king-lizard.md
│   ├── gigazaur.md
│   └── meka-dragon.md
├── build.py              ← Build script: converts .md files → data.json
├── data.json             ← Generated by build.py (don't edit manually)
├── index.html            ← The web app (no framework, just HTML/CSS/JS)
├── package.json          ← Optional: for node-based build (advanced)
└── README.md             ← This file
```

## Workflow: Adding or Editing a Monster

### 1. Create (or edit) a Markdown file in `/monsters`

Copy an existing file and rename it (e.g. `cyber-bunny.md`). The filename becomes the monster's ID.

Each file has two parts:

**Frontmatter** (between the `---` lines) — structured data:
```yaml
---
name: Cyber Bunny
icon: 🐰
tagline: "Fast, sneaky, impossible to pin down."
difficulty: Intermediate       # Beginner | Intermediate | Advanced
playstyle: Hit & Run / VP Racer
stats:
  aggression: 3
  resilience: 2
  energy: 4
  speed: 5
---
```

**Body** — free-form Markdown (headings, lists, tables, bold text, etc.):
```markdown
## Overview
Cyber Bunny is all about speed...

## Tips
- Roll lightning bolts early...
```

### 2. Run the build script

```bash
python3 build.py
```

This reads all `.md` files and generates `data.json`. You must do this every time you change a monster file.

### 3. Open `index.html` in your browser to preview

Just double-click it, or use a local server:
```bash
python3 -m http.server 8080
# then open http://localhost:8080
```

### 4. Commit and push to GitHub

GitHub Pages will serve your `index.html` automatically.
Make sure `data.json` is committed too (it's required at runtime).

## Monster File Reference

### Required frontmatter fields
| Field       | Example                          | Notes                        |
|-------------|----------------------------------|------------------------------|
| `name`      | `King Lizard`                    | Display name                 |
| `icon`      | `🦎`                             | Emoji shown in sidebar       |
| `tagline`   | `"The classic destroyer."`       | Short one-liner              |
| `difficulty`| `Beginner`                       | Beginner / Intermediate / Advanced |
| `playstyle` | `Tank / Tokyo Camper`            | Short descriptor             |
| `stats`     | see example above                | aggression/resilience/energy/speed, 1–5 |

### Markdown body sections (suggested, not required)
- `## Overview` — who the monster is
- `## Attributes` — table of strengths/weaknesses  
- `## Playstyle` — how they win
- `## Tips & Strategies` — with `### Early Game`, `### Mid Game` etc.
- `## Card Priorities` — numbered list
- `## Common Mistakes` — bullet list

## Deployment (GitHub Pages)

1. Push this entire folder to a GitHub repository
2. Go to **Settings → Pages**
3. Set source to **Deploy from branch** → `main` → `/root`
4. Your site will be live at `https://yourusername.github.io/your-repo-name/`

> **Remember:** run `python3 build.py` and commit `data.json` before pushing!

## Customisation Ideas

- Add a `rating` or `fun_factor` field to frontmatter and display it
- Add a `expansion` field to tag which box a monster comes from
- Add monster artwork images to `/images/` and reference them in frontmatter
- Add a "compare monsters" side-by-side view
